# Воркшоп «Мониторинг и логирование: Kibana, Grafana, ElasticSearch»
### Инструкция по подготовке рабочего места

Привет! Рады видеть тебя на воркшопе. Чтобы мы не теряли время на установку прямо во время занятия — пожалуйста, подготовь рабочее место **заранее**.

Вся установка займёт **15–20 минут**. Если что-то пойдёт не так — напиши нам до воркшопа, разберёмся вместе.

---

## Шаг 1. Проверь системные требования

| Параметр | Минимум | Рекомендуется |
|----------|---------|---------------|
| ОЗУ | 8 GB | 16 GB |
| CPU | 4 ядра | 4+ ядра |
| Свободное место на диске | 15 GB | 20 GB |
| Операционная система | Windows 10/11, macOS 12+, Ubuntu 20.04+ | macOS или Linux |

> ⚠️ **Windows-пользователям:** потребуется WSL2 (подробнее ниже). Убедись, что виртуализация включена в BIOS.

---

## Шаг 2. Установи Docker Desktop

### 🍎 macOS (Apple Silicon и Intel)

1. Перейди на сайт: **https://docs.docker.com/desktop/install/mac-install/**
2. Скачай и установи Docker Desktop под свою архитектуру (Apple Silicon или Intel)
3. Запусти Docker Desktop и дождись, пока иконка в трее станет зелёной

### Windows 10 / 11

**Сначала установи WSL2:**

Открой PowerShell от имени администратора и выполни:
```powershell
wsl --install
```
Перезагрузи компьютер после установки.

**Затем установи Docker Desktop:**

1. Перейди на сайт: **https://docs.docker.com/desktop/install/windows-install/**
2. Скачай и запусти установщик
3. При установке отметь галочку **"Use WSL 2 based engine"**
4. Запусти Docker Desktop и дождись зелёной иконки в трее

### Ubuntu / Debian

Открой терминал и выполни по шагам:

```bash
# Удалить старые версии (если есть)
sudo apt-get remove docker docker-engine docker.io containerd runc

# Установить зависимости
sudo apt-get update
sudo apt-get install -y ca-certificates curl gnupg

# Добавить GPG-ключ Docker
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# Добавить репозиторий
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" \
  | sudo tee /etc/apt/sources.list.d/docker.list

# Установить Docker
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# Запускать Docker без sudo
sudo usermod -aG docker $USER
newgrp docker
```

## Шаг 3. Проверь установку

Открой терминал (или PowerShell) и выполни:

```bash
docker --version
docker compose version
docker run --rm hello-world
```

**Ожидаемый результат:**
```
Docker version 26.x.x, build ...
Docker Compose version v2.x.x
Hello from Docker!   ← это сообщение означает, что всё работает
```

## Шаг 4. Настрой лимит виртуальной памяти (обязательно для Linux и Windows WSL2)

Elasticsearch требует повышенного лимита `vm.max_map_count`. Без этого он не запустится.

### Linux:
```bash
# Применить сейчас
sudo sysctl -w vm.max_map_count=262144

# Сделать постоянным (после перезагрузки)
echo "vm.max_map_count=262144" | sudo tee -a /etc/sysctl.conf
sudo sysctl -p
```

### Windows (WSL2) — выполни в PowerShell от администратора:
```powershell
wsl -d docker-desktop sysctl -w vm.max_map_count=262144
```

> 💡 **macOS-пользователям** этот шаг не нужен — Docker Desktop на Mac управляет этим автоматически.

## Шаг 5. Скачай образы заранее

На площадке воркшопа может быть медленный интернет. Пожалуйста, скачай образы **заранее** — это ~3 GB:

```bash
docker pull docker.elastic.co/elasticsearch/elasticsearch:8.11.0
docker pull docker.elastic.co/kibana/kibana:8.11.0
docker pull docker.elastic.co/logstash/logstash:8.11.0
docker pull grafana/grafana:latest
docker pull prom/prometheus:latest
docker pull prom/node-exporter:latest
```

Скачивание займёт 5–10 минут при хорошем соединении. Когда все образы загрузятся, ты увидишь их командой:

```bash
docker images
```

## Шаг 6. Финальная проверка готовности

Выполни эту команду — она проверит всё сразу:

```bash
docker --version && \
docker compose version && \
docker run --rm hello-world && \
echo "" && \
echo "🎉 Отлично! Всё готово к воркшопу."
```

Если в конце видишь `🎉 Отлично! Всё готово к воркшопу.` — ты полностью готов.
